# 스토리시드 — 데스크톱 앱 만들기 안내

이 폴더는 「스토리시드」를 맥(.dmg)과 윈도우(.exe/.msi) 설치 파일로 만들 수 있는 Tauri 프로젝트예요.
직접 빌드하는 방법(A)과, GitHub에게 대신 빌드시키는 방법(B) 두 가지가 있어요.
**배포가 목적이라면 B를 추천해요** — 내 컴퓨터가 맥이어도 윈도우용까지 한 번에 나오거든요.

---

## A. 내 컴퓨터에서 직접 실행/빌드하기

### 1회만 하면 되는 준비
1. **Node.js 설치** — https://nodejs.org 에서 LTS 버전 설치
2. **Rust 설치** — https://rustup.rs 접속 → 안내대로 설치 (터미널에 한 줄 붙여넣기)
   - 윈도우라면 설치 중 "Visual Studio C++ Build Tools"를 함께 설치하라고 안내가 떠요. 그대로 따라주세요.
3. 터미널(맥: 터미널 앱 / 윈도우: PowerShell)에서 이 폴더로 이동한 뒤:
   ```
   npm install
   npx tauri icon app-icon.png
   ```
   (아이콘 명령은 앱 아이콘 파일들을 자동 생성해요. 딱 한 번만.)

### 개발 모드로 열어보기
```
npm run dev
```
스토리시드가 데스크톱 앱 창으로 뜹니다. 코드를 고치면 바로 반영돼요.

### 설치 파일 만들기
```
npm run build
```
완성 파일 위치:
- 맥: `src-tauri/target/release/bundle/dmg/` 안의 .dmg
- 윈도우: `src-tauri/target/release/bundle/nsis/` 안의 .exe

> 맥에서는 맥용만, 윈도우에서는 윈도우용만 만들어져요. 양쪽 다 필요하면 B 방법을 쓰세요.

---

## B. GitHub Actions로 맥+윈도우 동시 빌드 (추천)

1. https://github.com 에 가입하고 **새 저장소(Repository)** 를 만들어요. (Private로 만들어도 돼요)
2. 이 폴더의 내용 전체를 저장소에 올려요.
   - 가장 쉬운 방법: GitHub Desktop 앱 설치 → 저장소 열기 → 이 폴더 내용 복사해 넣고 Commit & Push
3. GitHub 저장소 화면에서 **Actions 탭** → "스토리시드 설치 파일 만들기" → **Run workflow** 버튼 클릭
4. 10~20분 뒤 완료되면 실행 내역 하단의 **Artifacts**에서
   `스토리시드-mac`(dmg)과 `스토리시드-windows`(exe/msi)를 내려받을 수 있어요.

## 처음 실행할 때 알아둘 것

- **맥**: 서명(공증) 없이 빌드된 앱이라 처음 열 때 "확인되지 않은 개발자" 경고가 떠요.
  앱을 **우클릭 → 열기 → 열기**를 한 번만 해주면 그다음부터는 그냥 열려요.
  (다른 사람에게 배포할 때도 이 안내를 함께 전해주세요. 정식 배포 단계가 되면
  Apple 개발자 계정($99/년)으로 서명/공증을 붙일 수 있어요.)
- **윈도우**: SmartScreen 경고가 뜨면 "추가 정보 → 실행"을 눌러주세요.
- 첫 실행에서 서재 폴더를 한 번 선택하면, 그다음부터는 **"최근 서재 바로 열기"** 버튼으로
  폴더 선택 없이 바로 이어서 쓸 수 있어요.
- 브라우저 버전(서재-v0.4.html)에서 쓰던 서재 폴더를 그대로 선택하면 원고가 전부 이어져요.

## 폴더 구조

```
seojae-tauri/
├── src/index.html        ← 앱 화면과 기능 전체 (여기만 고치면 앱이 바뀌어요)
├── src-tauri/            ← Tauri 껍데기 (Rust) — 건드릴 일 거의 없음
├── app-icon.png          ← 앱 아이콘 원본 (교체 가능, 1024×1024 PNG)
└── .github/workflows/    ← GitHub 자동 빌드 설정
```
