# 실전: 자동 업데이트 실제로 작동시키기 (2.0.0 → 2.0.1)

자동 업데이트는 "**이전 버전이 설치돼 있어야**" 새 버전으로 갱신됩니다.
즉, 첫 릴리스(2.0.0)를 한 번 내보내 설치해두고, 그다음 2.0.1을 올리면 그때 자동 업데이트가 동작합니다.
지금 드린 코드는 이미 **2.0.1**이라, 아래 순서대로 하시면 됩니다.

---

## A. 딱 한 번만: 서명키 + 시크릿 (5분)

`자동업데이트-설정-가이드.md`의 1~3단계와 동일합니다. 요약:

1. 터미널에서 서명키 생성(무료):
   ```
   npm run tauri signer generate -- -w ~/.tauri/storyseed.key
   ```
2. 출력된 **공개키**를 `src-tauri/tauri.conf.json`의 `"pubkey"`에 붙여넣기.
   같은 파일 `"endpoints"`의 `GITHUB_ID/REPO`를 실제 저장소로 교체.
3. GitHub 저장소 → Settings → Secrets and variables → Actions 에 2개 등록:
   - `TAURI_SIGNING_PRIVATE_KEY` = `~/.tauri/storyseed.key` 파일 내용 전체
   - `TAURI_SIGNING_PRIVATE_KEY_PASSWORD` = 생성 때 정한 비밀번호(없으면 빈 값)

> 이 3가지가 없으면 빌드는 되지만 자동 업데이트 서명이 빠져 갱신이 안 됩니다.

---

## B. 첫 배포 (2.0.0을 "기준점"으로 한 번)

자동 업데이트를 시험하려면 먼저 **더 낮은 버전**이 설치돼 있어야 해요.
가장 확실한 방법은 2.0.0을 먼저 올리는 것입니다.

1. 네 파일의 버전을 **2.0.0**으로 잠깐 내려요: `package.json`, `src-tauri/tauri.conf.json`,
   `src-tauri/Cargo.toml`, `src/index.html`의 `APP_VERSION`.
   (지금은 2.0.1이니, 첫 배포만 2.0.0으로 낮췄다가 곧 다시 2.0.1로 올릴 거예요.)
2. 커밋 후 태그 push:
   ```
   git add -A && git commit -m "v2.0.0"
   git tag v2.0.0 && git push && git push origin v2.0.0
   ```
3. GitHub **Actions → 스토리시드 릴리스 (자동 업데이트)** 완료까지 대기(10~20분).
4. Release에서 **windows 설치본(.exe)**을 내려받아 **설치**하고 한 번 실행해 둡니다.
   → 이제 내 PC에 "2.0.0"이 깔린 상태.

## C. 업데이트 배포 (2.0.1)

1. 네 파일의 버전을 다시 **2.0.1**로 올립니다(지금 드린 상태 그대로면 이미 2.0.1).
2. 태그 push:
   ```
   git add -A && git commit -m "v2.0.1"
   git tag v2.0.1 && git push && git push origin v2.0.1
   ```
3. Actions 완료 후, **아까 설치해둔 2.0.0 앱을 실행**해 보세요.
   → 몇 초 뒤 "새 버전 2.0.1이 나왔어요" 안내가 뜨고,
     확인을 누르면 자동으로 내려받아 설치·재시작됩니다.
   → 환경설정 → 정보 → **지금 확인**으로 즉시 확인도 가능.

이게 되면 자동 업데이트가 완전히 작동하는 겁니다. 이후로는 **C 단계만 반복**(버전 올리고 태그 push)하면 돼요.

---

## 앞으로의 평소 루틴 (외워두면 3줄)

```
1) 네 파일 버전 올리기 (2.0.2 …)
2) git tag v2.0.2 && git push origin v2.0.2
3) 20분 뒤 끝 — 사용자는 앱 켜면 자동 업데이트
```

## 점검 체크리스트

- [ ] `tauri.conf.json`의 `pubkey`가 실제 공개키인가? (`PASTE_...` 그대로면 안 됨)
- [ ] `endpoints`의 `GITHUB_ID/REPO`를 실제 값으로 바꿨는가?
- [ ] GitHub Secrets에 개인키 2개가 등록됐는가?
- [ ] Release에 `latest.json`이 올라왔는가? (Actions 성공 후 Release 자산 목록에서 확인)
- [ ] 네 곳의 버전 번호가 서로 같은가?

## 자주 막히는 곳

- **"업데이트가 안 떠요"** → 설치된 버전이 최신과 같거나 높으면 안 떠요. 반드시 낮은 버전이 깔려 있어야 합니다(B 단계).
- **"서명 오류"** → pubkey(공개키)와 Secrets의 개인키가 **같은 쌍**인지 확인.
- **"latest.json 404"** → 아직 Release가 없거나 endpoints의 저장소 경로가 틀림.
- **비용** → 전 과정 무료입니다(GitHub Releases + Tauri 서명).
