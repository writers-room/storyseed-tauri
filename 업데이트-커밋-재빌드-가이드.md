# 수정된 파일을 GitHub에 올리고 다시 빌드하기 — 상세 가이드

이번 v0.6 업데이트에서 바뀐 파일은 **7개**예요. 아래 순서대로 하면 10분이면 끝나요.

## 이번에 바뀐 파일 목록

| 파일 | 바뀐 내용 |
|---|---|
| `src/index.html` | 앱 기능 전체 (거의 모든 수정이 이 파일이에요) |
| `src-tauri/tauri.conf.json` | 앱 이름 StorySeed, 한국어 설치 화면 |
| `src-tauri/capabilities/default.json` | 내보내기용 저장 권한 |
| `src-tauri/src/main.rs` | 오류 문구 |
| `src-tauri/Cargo.toml` | 설명 문구 |
| `.github/workflows/build.yml` | 결과물 이름 |
| `app-icon.png` | 새 아이콘 (씨앗+깃펜) |

> 앞으로의 업데이트는 대부분 `src/index.html` **한 개만** 바꾸면 돼요.

## 1단계: 새 zip 준비

1. 받은 `스토리시드-tauri-v0.6.zip` 압축을 풀어요 → `storyseed-tauri` 폴더가 생겨요.
2. (맥) 폴더 안에서 `Cmd+Shift+.` 눌러 숨김 파일 보이게 하기 — 지난번과 동일해요.

## 2단계: GitHub에 덮어쓰기 업로드

GitHub는 **같은 이름의 파일을 올리면 자동으로 새 버전으로 덮어써요.** 지우고 다시 올릴 필요가 없어요.

1. github.com 접속 → 로그인 → 내 저장소(`seojae`) 클릭
2. 오른쪽 위 **Add file** 버튼 → **Upload files** 클릭
3. 압축 푼 `storyseed-tauri` 폴더 안의 **내용물 전부 선택**(`Cmd+A`/`Ctrl+A`)
   → 브라우저 업로드 영역으로 드래그
   - 폴더째 끌면 안 되고, 폴더를 열고 그 안의 것들을 끌어야 해요 (지난번과 동일)
4. 아래 "Commit changes" 칸에 메모를 남기면 좋아요. 예: `v0.6 — 스토리시드로 이름 변경, 내보내기 추가`
   (이 메모가 바로 "커밋 메시지"예요. 나중에 언제 뭘 바꿨는지 확인하는 용도)
5. 초록색 **Commit changes** 버튼 클릭 → 업로드 완료까지 대기

## 3단계: 다시 빌드

1. 저장소 상단 **Actions 탭** 클릭
2. 왼쪽 목록에서 **스토리시드 설치 파일 만들기 (mac + windows)** 클릭
   - 이름이 아직 "서재 설치 파일 만들기"로 보이면 새로고침을 한 번 해주세요.
3. **Run workflow** → 초록색 **Run workflow** 클릭
4. 10~20분 후 초록 체크✓ → 해당 실행 클릭 → 아래 **Artifacts**에서
   `스토리시드-mac` / `스토리시드-windows` 내려받기
5. 설치하면 이름과 아이콘이 바뀐 「스토리시드」를 확인할 수 있어요.

## 자주 하게 될 "평소의 업데이트" (외워두면 좋은 3줄 요약)

1. 새로 받은 `index.html`을 → 저장소 **Add file → Upload files**로 `src` 폴더 위치에 드래그 → Commit
   - 주의: `src/index.html` 경로가 유지되어야 해요. 업로드 화면에서 파일을 끌기 전에
     저장소의 `src` 폴더를 먼저 클릭해서 들어간 다음 Add file → Upload files 하면 정확해요.
2. **Actions → Run workflow**
3. 20분 뒤 **Artifacts** 다운로드

## 참고: 기존 원고는 안전한가요?

네. GitHub에 올라가는 건 **프로그램 코드뿐**이고, 원고는 내 컴퓨터의 작품 폴더에만 있어요.
새 버전을 설치해도 기존 작품 폴더를 "최근 작품 바로 열기"로 열면 모든 원고·메모·트리트먼트가 그대로예요.
